# Ethics and Data Availability Notes

## Ethics

- The dashboard is designed for screening support, not diagnosis.
- User-facing consent should be obtained before data capture.
- Sensitive records should be de-identified before any publication screenshots or tables are shared.
- Demo or bundled sample records should never be treated as real patient data in the paper.

## Data Availability

- Source code: project repository.
- Training or evaluation data: describe the exact access conditions and permissions used in the study.
- Any locally generated figures or tables should be reproducible from the saved evaluation outputs.

## Reproducibility Checklist

- State the software version and environment used.
- List the dataset roots or manifests used for the final experiments.
- Include the exact evaluation command and output location.
- Document any proxy metrics or fallback heuristics separately from the main clinical model results.
